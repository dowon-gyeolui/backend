# DOMAIN_LOGIC.md

## 1. Domain Overview
이 프로젝트는 사주 기반 인연 매칭 서비스다.
도메인 특성상 해석 요소가 많지만,
현재 MVP에서는 완벽한 명리학 구현보다
서비스 흐름 검증에 필요한 최소 도메인 구조만 반영한다.

---

## 2. Key Concepts

### 사주
사용자의 생년월일, 태어난 시간, 양력/음력 여부, 성별, 윤달 여부를 기반으로 계산되는 기본 운세 데이터.

### 자미두수
사주와는 별도의 해석 체계.
현재 MVP에서는 유료 기능 카테고리로만 두고,
실제 계산은 placeholder 처리 가능하다.

### 궁합 점수
두 사용자 간의 매칭 적합도를 수치화한 값.
현재 MVP에서는 0~100 점 범위의 값으로 처리한다.

### 매칭 후보
궁합 점수와 기본 조건을 바탕으로 추천되는 상대 사용자.

### 블라인드 프로필
매칭 직후에는 일부 정보만 공개된 상태의 프로필.

### 확률 높이는 방법
매칭 성사 가능성을 높이기 위해 제공하는 추천 정보.
예시:
- 추천 색상
- 추천 장소
- 추천 스타일링
- 행동 가이드

---

## 3. MVP Logic Assumptions
현재 MVP는 아래 가정을 따른다.

1. 사주 계산은 외부 전문 엔진이 없어도 임시 구조로 진행 가능하다.
2. 궁합 계산은 rule-based 로직으로 시작한다.
3. 점수의 절대 정확성보다 일관된 사용자 경험이 우선이다.
4. 추천 문구는 AI 생성 또는 템플릿 방식 중 단순한 방식으로 시작 가능하다.

---

## 4. Basic Compatibility Score Rule
초기 궁합 점수 계산은 예시 수준으로 아래처럼 단순화할 수 있다.

- 기본 점수: 50
- 특정 조건 충족 시 가산점
- 특정 조건 불일치 시 감점
- 최종 점수는 0~100 사이로 제한

실제 상세 명리학 로직은 후속 작업에서 대체 가능하다.

중요:
현재 단계에서는 임의의 복잡한 점술 체계를 구현하지 않는다.

## 5. Paid Feature Logic
유료 기능은 아래처럼 분리한다.

- 자미두수 결과 조회
- 상세 궁합 해석
- 추가 후보 열람
- 확률 높이는 방법의 상세 결과

초기 MVP에서는 결제 여부를 나타내는 권한 플래그만으로 처리 가능하다.
