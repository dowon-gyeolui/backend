# DOMAIN_LOGIC.md

ZAMI 의 도메인 용어와 핵심 비즈니스 규칙. 작업 시 용어 정의의 단일 원천.

---

## 1. Domain Overview

사주 + 자미두수 기반 한국 데이팅 서비스. 두 가지 핵심 가치를 제공:

1. 두 사용자 간의 명리학적 적합도 계산 (사주 궁합 점수)
2. 매칭 성공 확률을 높이는 행동 / 추천 (오늘의 인연운 · 행동 가이드 · 데이트 코스 · 운명의 실타래 등)

---

## 2. Key Concepts

### 사주
사용자의 생년월일 + 출생 시간 + 양력/음력 + 성별 + 윤달 여부를 입력으로
정통 60갑자 알고리즘으로 산출되는 4기둥(년주/월주/일주/시주). 절기 기준
월주, `1900-01-31(갑진일)` 기준 일주, 五虎遁/五鼠遁 으로 월간/시간이
유도된다. 구현: `services/saju_engine.py`.

### 자미두수
사주와 별도의 명리학 체계. 12궁 × 14주성 × 4 사화 × 부성/살성을 표준
안성술(安星術)로 계산. 구현: `services/jamidusu/chart.py`. 차트는 결정론적
이며, LLM 풀이는 별도 단계로 grounding 된다.

### 궁합 점수
두 사용자 간 명리학적 적합도. 0~100 정수. 일간(日干) 오행 관계 + 일지
삼합/육충 + 주도 오행 상생 관계로 점수화. rule-based MVP 구현
(`services/compatibility.py`). 정확도보다는 입력에 따라 의미 있게 변하는
안정 시그널이 목표.

### 4-슬롯 daily 매칭
한 사이클(96시간) 당 4명의 후보가 슬롯 0~3 에 배정된다. 모든 사용자가
KST 자정 00:00 anchor 를 공유하므로 슬롯 unlock 시각이 사용자별로
달라지지 않는다. 슬롯 1·3 은 자미두수 기반 + 유료 (결제 시 사진 공개),
슬롯 0·2 는 사주 기반 + 무료.

### 누적 매칭 히스토리
`/compatibility/history` 가 반환하는, 그동안 노출됐던 모든 후보 (cycle
구분 없이 candidate_id dedup). 매칭 리스트 화면의 무한 스크롤 데이터.

### 블라인드 프로필
무료 티어 사용자에게 노출되는 일부 정보만 공개된 매칭 카드. 사진은 blur
처리 (백엔드에서 photo_url 자체를 None 으로 내려보냄). 결제 후 unblind.

### 행동 가이드 / 인연운
사용자 사주 + 오늘 일진(日辰) 의 십성·합·충 관계로 매일 동적으로 생성되는
3줄 산문. LLM 호출 없음, 결정론적 룩업 + 템플릿. 구현:
`services/action_guide.py` + `services/fortune.py`.

### 운명의 실타래 / 데이트 코스
두 사용자 사주를 비교해 5섹션(첫인상/성격/연애 스타일/주의/장기 전망) 풀이
또는 4~5개 데이트 장소를 LLM 으로 생성. 유료. 구현:
`services/llm/interpret.py` + `routers/compatibility.py`.

### 모더레이션 스트라이크
채팅 메시지가 3-layer 모더레이션 (정규식 / 욕설 사전 / OpenAI Moderation)
에 차단될 때 누적되는 위반 기록. 24h 내 3회 누적 시 `chat_suspended_until`
이 24h 미래로 설정되어 채팅이 일시 정지된다.

### 얼굴 인증
프로필 메인 사진이 AWS Rekognition strict 검수(얼굴 1개 + 면적 25% 이상)를
통과한 경우. `is_face_verified=True`. 이 사진에만 ZAMI 공식 인증 뱃지가
노출된다.

## 3. 궁합 점수 산정 규칙

현재 rule-based 모델 (`services/compatibility.py`):

```
base                                = 50
일간(日干) 오행 관계                = +5 / +10 / -10 / 0
일지(日支) 삼합(三合)               = +10
일지(日支) 육충(六沖)               = -10
주도 오행 상생 관계                 = +5
최종 점수는 0~100 으로 clamp
```

정확도보다는 입력에 따라 의미 있게 변하는 안정 시그널을 우선. 추후 RAG
기반 근거 인용을 score 응답에 attach 하는 것이 TODO.

## 4. 유료 기능 분리

`is_paid` 플래그 하나로 다음을 분기:

- 자미두수 풀이 열람 (`/saju/me/jamidusu-deep`)
- 운명의 실타래 5섹션 풀이 (`/compatibility/destiny/{id}`)
- 데이트 코스 추천 (`/compatibility/date-recommendation/{id}`)
- 4-슬롯 중 자미두수 슬롯(1, 3) 의 사진 unblind
- 채팅방 입장 권한

결제 시스템은 PortOne / 토스페이먼츠 연동 예정. 그 전까지는 데모
모드에서 `/users/me/upgrade-demo` 가 `is_paid=True` 토글.

## 5. 의도적으로 Out of Scope

다음 기능들은 ZAMI 의 정체성과 무관하므로 의도적으로 구현하지 않는다.
사용자 요청이 와도 이쪽으로 확장하지 않는다.

- 작명소
- 관상 (face reading) — Rekognition 사용은 NSFW/얼굴 검수 용도이지 관상 아님
- 풍수지리 고도화
- 캐릭터 시스템
- 복잡한 소셜 피드 / 커뮤니티
- 대규모 관리자 대시보드
- 다국어 지원 (한국어 전용)
