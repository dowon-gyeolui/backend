# CURRENT_MVP_SCOPE.md

## 1. Purpose
이 문서는 현재 개발해야 하는 MVP 범위를 명확히 제한하기 위한 문서다.

Claude Code는 이 문서를 기준으로
"지금 구현해야 하는 것"과
"지금 구현하지 말아야 하는 것"을 반드시 구분해야 한다.

---

## 2. In Scope
현재 MVP에서 구현 대상인 기능은 아래와 같다.

### Required
1. 카카오 로그인
2. 사용자 기본 정보 저장
   - 생년월일
   - 태어난 시간
   - 양력/음력 여부
   - 성별
   - 윤달 여부
3. 사용자 사주 기본 계산 결과 저장 또는 반환
4. 기본 궁합 점수 계산 로직
5. 매칭 후보 2명 이상 반환하는 기능
6. 무료/유료 기능 구분을 위한 데이터 구조
7. "확률 높이는 방법"에 대한 기본 추천 구조
   - 색상 추천
   - 장소 추천
   - 스타일링 추천 문구

### Optional but Allowed
- 자미두수 기능은 구조만 잡고 실제 계산은 placeholder 처리 가능
- 결제 기능은 실제 PG 연동 없이 상품/권한 상태 모델만 구현 가능
- 채팅 기능은 실제 구현 없이 "채팅 가능 상태"까지의 데이터 구조만 설계 가능

---

## 3. Out of Scope
현재 MVP에서 구현하지 않는 기능은 아래와 같다.

1. 작명소 기능
2. 붓글씨 생성
3. 관상 분석
4. 풍수지리 고도화 분석
5. 캐릭터 시스템
6. 정교한 이미지 분석
7. 실시간 추천 AI 파이프라인
8. 복잡한 소셜 피드
9. 고도화된 커뮤니티 기능
10. 고급 데이터 대시보드
11. 관리자 페이지 전체 구현
12. 정교한 ML 모델 학습
13. 대규모 트래픽 대응 최적화
14. 다국어 지원
15. 정교한 추천 알고리즘 실험

---

## 4. Implementation Principle
현재 MVP는 아래 원칙으로 구현한다.

- 복잡한 도메인 해석보다 동작 가능한 최소 기능을 우선한다.
- 계산 로직은 처음부터 완벽할 필요가 없다.
- 우선은 rule-based 로직으로 시작한다.
- 실제 외부 연동이 어려운 기능은 placeholder나 mock으로 처리 가능하다.
- 중요한 것은 "흐름이 이어지는 제품"을 만드는 것이다.

---

## 5. Decision Rule
새로운 기능 요청이 들어왔을 때 아래 기준으로 판단한다.

### 구현한다
- 현재 핵심 사용자 흐름을 완성하는 기능
- 로그인 → 정보 입력 → 결과 확인 → 매칭 확인 흐름에 직접 필요한 기능
- 무료/유료 구분에 직접 필요한 기능

### 구현하지 않는다
- 재미 요소이지만 핵심 흐름과 무관한 기능
- 시간이 많이 드는 고급 기능
- 외부 서비스 연동이 큰 기능
- 현재 핵심 검증과 관계없는 기능

---

## 6. If Unclear
요구사항이 애매할 경우 아래 기준을 따른다.

1. 현재 MVP 흐름에 직접 필요하면 최소 수준으로 구현한다.
2. 현재 MVP 흐름과 무관하면 구현하지 않는다.
3. 확장 아이디어는 TODO 또는 NOTE로만 남기고 실제 구현하지 않는다.
